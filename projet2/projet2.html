Présentation du Projet

Dans le cadre de mon BTS SIO option SLAM, j'ai développé une application web PHP mettant en évidence plusieurs failles de sécuritées comme Injection SQL, faille XSS, CSRF, Failles d’authentification, Directory traversal.

Durée : 1 mois
Équipe : 1 développeur (moi meme)
Statut : finalisé

Technologies Vulnérables

PHP (sans framework)

  • Accès direct à $_GET/$_POST sans validation
  • Pas de PDO préparé (injection SQL possible)
  • Include dynamique (LFI/RFI)

MySQL/MariaDB

  • Requêtes SQL concaténées non sécurisées
  • Mots de passe en md5() ou clair
  • Pas de contrôle des permissions

Front-end

  • Formulaires sans validation JS
  • Absence de CSP (XSS possible)
  • innerHTML non sécurisé

Gestion des fichiers

  • Upload sans vérification d'extension
  • Dossiers accessibles en exécution
  • Directory traversal possible

Communication HTTP

  • Pas de HTTPS (données en clair)
  • Headers de sécurité manquants
  • Cookies sans Secure/HttpOnly

Authentification

  • Pas de bcrypt (hash faible)
  • Sessions non régénérées
  • Brute-force possible

Bonnes pratiques à implémenter

  • Utiliser PDO avec requêtes préparées
  • Implémenter password_hash()/password_verify()
  • Ajouter Content-Security-Policy
  • Valider/sanitizer toutes les entrées

Analyse Sécurité & Améliorations

Vulnérabilités Identifiées

  • Implémenter l'échappement XML et la validation XSD
  • Mise en place de htmlspecialchars() pour contrer XSS
  • Migration vers bcrypt pour le stockage mots de passe
  • Ajout de tokens CSRF sur les formulaires
  • Implémentation de Content-Security-Policy
  • Journalisation des activités suspectes
